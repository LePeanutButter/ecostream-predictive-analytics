# EcoStream: Contexto Técnico Actualizado

## Arquitectura

- **Patrón:** Clean Architecture.
- **Flujo:** Controller -> Use Case -> Domain Service -> Repository.

## Entidades Core

- `ActividadTransporte`: Atributos (tipo_vehiculo, distancia_km, peso_toneladas, factor_eficiencia).
- `FactorEmision`: Atributos (tipo_vehiculo, kg_co2e_por_km).
- `ResultadoHuella`: Calcula automáticamente toneladas a partir de kg.

## Stack de Pruebas

- **Herramienta:** Pytest.
- **Estrategia:** Mocks de repositorios para lógica de dominio y pruebas de integración para Use Cases.

## Excepciones Definidas

- `ValorInvalidoException` (422 Unprocessable Entity).
- `TipoVehiculoNoSoportadoException` (400 Bad Request).
